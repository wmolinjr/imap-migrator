Migrador de Caixas de Correio IMAP
Este módulo permite migrar caixas de correio de um servidor IMAP para outro usando a ferramenta imapsync. Ele fornece uma interface amigável para configurar e executar migrações de caixas de correio com monitoramento de progresso em tempo real.

Pré-requisitos

Antes de usar este módulo, certifique-se de que:
  • A ferramenta imapsync está instalada no seu sistema
  • Você tem credenciais de acesso para ambos os servidores IMAP de origem e destino
  • Há conectividade de rede entre seu servidor e ambos os servidores IMAP

Aba: Credenciais

Esta é a aba principal de configuração onde você define os detalhes de conexão: Servidor de Origem:
  • Servidor IMAP: O nome do host ou endereço IP do seu servidor IMAP de origem
  • Usuário: O nome de usuário da caixa de correio de origem
  • Senha: A senha da caixa de correio de origem
  • Segurança: Escolha entre Nenhum, SSL ou TLS para a conexão
Servidor de Destino:
  • Servidor IMAP: O nome do host ou endereço IP do seu servidor IMAP de destino (padrão: localhost)
  • Usuário: O nome de usuário da caixa de correio de destino
  • Senha: A senha da caixa de correio de destino
  • Segurança: Escolha entre Nenhum, SSL ou TLS para a conexão

Aba: Avançado

Opções avançadas de configuração para a migração:
  • Tamanho máximo: Limitar o tamanho das mensagens em MB
  • Tamanho mínimo: Tamanho mínimo das mensagens em MB
  • Idade máxima: Pular mensagens mais antigas que os dias especificados
  • Idade mínima: Pular mensagens mais novas que os dias especificados
  • Tipo de seleção por idade: Escolha como combinar os filtros maxage e minage:
    • Padrão: Usar maxage ou minage independentemente
    • Intersecção: Mensagens devem estar dentro do intervalo maxage E minage
    • União: Mensagens podem estar dentro do intervalo maxage OU minage
  • Excluir da origem: Remover mensagens do servidor de origem após migração bem-sucedida
  • Pular pastas vazias: Pastas vazias da origem não são criadas no destino
  • Timeout de conexão da origem: Timeout para conexões do servidor de origem em segundos (padrão: 120)
  • Timeout de conexão do destino: Timeout para conexões do servidor de destino em segundos (padrão: 120)
  • Enviar relatório por email na origem: Escolha se deseja enviar o relatório final da migração para a INBOX da caixa de correio de origem (padrão: Não)
  • Enviar relatório por email no destino: Escolha se deseja enviar o relatório final da migração para a INBOX da caixa de correio de destino (padrão: Sim)
  • Incluir pastas: Especifique padrões de pastas para incluir (ex: "INBOX,Sent,Drafts")
  • Excluir pastas: Especifique padrões de pastas para excluir (ex: "Trash,Spam")
  • Pular erros: Continuar a migração mesmo se algumas mensagens falharem

Aba: Depurar

Opções de depuração e log:
  • Debug do Migrador: Habilita o debug do migrador
  • Modo debug: Habilitar informações de debug
  • Apenas conectar aos servidores: Apenas conectar aos dois servidores e imprimir informações úteis
  • Modo de teste: Modo de teste - mostra o que seria migrado sem realmente fazer
  • Mostrar comandos: Exibir os comandos imapsync sendo executados
  • Debug de pastas: Modo debug apenas para a parte de pastas
  • Debug de conteúdo das mensagens: Debug do conteúdo das mensagens transferidas (saída enorme)
  • Debug de flags: Modo debug para flags
  • Debug IMAP do servidor de origem: Modo debug IMAP para host1 (muito verboso)
  • Debug IMAP do servidor de destino: Modo debug IMAP para host2 (muito verboso)
  • Debug de consumo de memória: Modo debug mostrando consumo de memória após cada cópia
  • Debug SSL: Modo debug SSL de 0 a 4

Processo de Migração

Depois de configurar todas as configurações:
  1. Clique no botão Iniciar Migração
  2. A migração será executada em background
  3. O progresso em tempo real será exibido na janela de log
  4. O log é atualizado automaticamente a cada 5 segundos
  5. A migração é concluída quando você vê "Exiting with return value"

Notas Importantes

  • Caixas de correio grandes podem levar tempo significativo para migrar
  • Certifique-se de espaço suficiente em disco em ambos os servidores
  • A estabilidade da rede é crucial para uma migração bem-sucedida
  • Considere usar SSL/TLS para conexões seguras
  • Teste com uma caixa de correio pequena primeiro usando o modo de teste
