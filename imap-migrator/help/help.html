IMAP Mailbox Migrator
This module allows you to migrate email mailboxes from one IMAP server to another using the imapsync tool. It provides a user-friendly interface to configure and execute mailbox migrations with real-time progress monitoring.

Prerequisites

Before using this module, ensure that:
  • The imapsync tool is installed on your system
  • You have access credentials for both source and destination IMAP servers
  • Network connectivity between your server and both IMAP servers

Tab: Credentials

This is the main configuration tab where you set up the connection details: Source Server:
  • IMAP Server: The hostname or IP address of your source IMAP server
  • User: The username for the source mailbox
  • Password: The password for the source mailbox
  • Security: Choose between None, SSL, or TLS for the connection
Destination Server:
  • IMAP Server: The hostname or IP address of your destination IMAP server (defaults to localhost)
  • User: The username for the destination mailbox
  • Password: The password for the destination mailbox
  • Security: Choose between None, SSL, or TLS for the connection

Tab: Advanced

Advanced configuration options for the migration:
  • Maximum size: Limit message size in MB
  • Minimum size: Minimum message size in MB
  • Maximum age: Skip messages older than specified days
  • Minimum age: Skip messages newer than specified days
  • Age selection type: Choose how to combine maxage and minage filters:
    • Default: Use maxage or minage independently
    • Intersection: Messages must be within both maxage AND minage range
    • Union: Messages can be within maxage OR minage range
  • Delete from source: Remove messages from source server after successful migration
  • Skip empty folders: Empty source folders are not created on destination
  • Source connection timeout: Timeout for source server connections in seconds (default: 120)
  • Destination connection timeout: Timeout for destination server connections in seconds (default: 120)
  • Send email report to source: Choose whether to send the final migration report to the source mailbox INBOX (default: No)
  • Send email report to destination: Choose whether to send the final migration report to the destination mailbox INBOX (default: Yes)
  • Include folders: Specify folder patterns to include (e.g., "INBOX,Sent,Drafts")
  • Exclude folders: Specify folder patterns to exclude (e.g., "Trash,Spam")
  • Skip errors: Continue migration even if some messages fail

Tab: Debug

Debugging and logging options:
  • Debug mode: Enable debug information
  • Just connect to servers: Only connect to both servers and print useful information
  • Dry run: Test mode - shows what would be migrated without actually doing it
  • Show commands: Display the imapsync commands being executed
  • Debug folders: Debug mode for the folders part only
  • Debug message content: Debug content of the messages transferred (huge output)
  • Debug flags: Debug mode for flags
  • Debug IMAP for source server: IMAP debug mode for host1 (very verbose)
  • Debug IMAP for destination server: IMAP debug mode for host2 (very verbose)
  • Debug memory consumption: Debug mode showing memory consumption after each copy
  • SSL debug: SSL debug mode from 0 to 4

Migration Process

Once you've configured all settings:
  1. Click the Start Migration button
  2. The migration will run in the background
  3. Real-time progress will be displayed in the log window
  4. The log updates automatically every 5 seconds
  5. Migration completes when you see "Exiting with return value"

Important Notes

  • Large mailboxes may take significant time to migrate
  • Ensure sufficient disk space on both servers
  • Network stability is crucial for successful migration
  • Consider using SSL/TLS for secure connections
  • Test with a small mailbox first using dry-run mode
